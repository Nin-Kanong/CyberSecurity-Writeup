- **This command performs a deep scan across your local subnet, trying to:
  ‚Ä¢ 	Detect live hosts
  ‚Ä¢ 	Identify open ports
  ‚Ä¢ 	Fingerprint services and versions
  ‚Ä¢ 	Guess the operating system
  ‚Ä¢ 	Run Nmap scripts for extra info (like vulnerabilities or misconfigurations)**
- Why It's Useful:
  - Recon in exam labs
  - Finding targets to exploit
  - Documenting host/service changes with `ndiff`
  - Building GitHub recon notes

<img width="1030" height="850" alt="image" src="https://github.com/user-attachments/assets/bba5a97c-ad11-46fa-af08-416729306d38" />


<img width="1022" height="862" alt="image" src="https://github.com/user-attachments/assets/ce3c7000-1319-4817-ae63-b573254d71f9" />


<img width="1010" height="861" alt="image" src="https://github.com/user-attachments/assets/20a7379d-60e6-4f91-97fe-5774df82b515" />


- `-sS`: TCP SYN scan (stealthy and fast)
- `-sU`: UDP scan (checks for services like DNS, SNMP, etc.).
- `-T4`: Aggressive timing for faster scanning.
- `-A`: Enables OS detection, version detection, script scanning, traceroute.
- `-v`: Verbose output (shows detailed progress).
- **This command:
  - Finds live hosts
  - Detects open TCP and UDP ports
  - Identifies services and versions
  - Guesses the operating system
  - Runs Nmap scripts to find vulnerabilities
  - Maps network paths with traceroute**
